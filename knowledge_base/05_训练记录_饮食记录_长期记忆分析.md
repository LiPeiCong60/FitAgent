# 训练记录、饮食记录与用户长期记忆分析

## 1. 为什么要记录

AI 健身助手的核心价值不是只回答通用知识，而是结合用户长期记录给出个性化判断。需要记录用户档案、身体状态、饮食日志、训练日志和对话记录。

## 2. 饮食记录设计

建议字段：
```json
{
  "user_id": "string",
  "date": "YYYY-MM-DD",
  "meal_type": "breakfast/lunch/dinner/snack",
  "foods": [
    {"name": "鸡胸肉", "amount_g": 200, "calories": 330, "protein_g": 62, "carbs_g": 0, "fat_g": 7}
  ],
  "total_calories": 0,
  "total_protein_g": 0,
  "total_carbs_g": 0,
  "total_fat_g": 0,
  "source": "manual/ai/image",
  "note": ""
}
```

## 3. 训练记录设计

建议字段：
```json
{
  "user_id": "string",
  "date": "YYYY-MM-DD",
  "session_type": "胸背/腿/推/拉/全身",
  "exercises": [

  ],

  "rpe": 8,
  "note": "状态不错"

```

## 4. 身体记录设计

建议字段：
```json
{
  "user_id": "string",
  "date": "YYYY-MM-DD",
  "weight_kg": 102,
  "body_fat_percent": 28,
  "waist_cm": 100,
  "lean_mass_kg": 73.4,
  "note": ""
}
```

## 5. AI 分析逻辑

饮食分析：最近 7 天平均热量、平均蛋白质、热量是否接近目标、蛋白质是否达标、哪些餐次最容易超标、是否经常漏记。训练分析：最近 7 天训练次数、每个部位训练频率、主要动作是否进步、是否连续高强度训练导致恢复不足、是否缺少腿部背部核心训练。身体变化分析：7 日平均体重变化、腰围变化、体脂趋势、是否需要继续观察而不是立刻调整。

## 6. 长期记忆与 RAG 的区别

长期记忆：读取用户自己的历史记录，例如吃了什么、练了什么、体重怎么变。RAG：从知识库检索外部知识，例如蛋白质建议、训练原则、动作注意事项。最佳方案是两者结合：用户记录提供个性化背景，RAG 提供知识依据。

## 7. Agent 工具设计

可设计工具：save_diet_log、save_workout_log、save_body_record、query_recent_diet、query_recent_workouts、query_body_trend、calculate_bmr、estimate_tdee、retrieve_knowledge。

## 8. 自动写入记录的规则

当用户说“今天吃了……”“午饭是……”“刚练完胸背……”“今天体重……”时，AI 可以判断为记录意图，调用对应工具写入数据库。为避免误写，建议先提取结构化数据；信息不完整时追问；写入后返回确认；允许用户修改或删除。

## 9. 示例回答片段

用户：我这周饮食怎么样？

回答参考：我先看你的最近 7 天记录。你的平均热量大约比目标高 250 kcal，主要来自晚餐和零食；蛋白质平均每天 135g，距离建议目标 160g 还有差距。建议先不大幅砍主食，而是优先减少晚餐油脂和零食，同时把每天蛋白质提高到 160g 左右。这样更有利于减脂期保留肌肉。
